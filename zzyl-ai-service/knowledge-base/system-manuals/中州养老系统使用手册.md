# 中州养老系统使用手册

## 系统概述

中州养老管理系统是一套面向养老机构的综合管理平台，涵盖老人入住管理、护理服务、床位管理、预约来访、订单财务等核心业务模块。系统集成RAG智能助手，为工作人员提供基于知识库的智能问答与操作导航服务。

## 系统架构

系统采用前后端分离架构：
- 前端：Vue3 + TypeScript + TDesign组件库
- 后端：Spring Boot + MyBatis + MySQL
- AI服务：Python LangChain + Chroma向量库 + 通义千问大模型

## 核心功能模块

### 入住管理

入住管理模块负责老人从申请到正式入住的全流程办理。

#### 功能入口

菜单路径：入住管理 > 入住办理

#### 主要功能

发起入住申请：为老人创建入住申请记录，填写老人基本信息、选择楼层房间床位、关联护理等级和合同信息。

分页查询：按老人姓名、身份证号查询入住申请列表，支持分页浏览。

查看详情：查看入住申请的完整信息，包括老人信息、床位信息、合同详情、护理等级等。

文件上传：支持上传入住相关文档，格式支持JPG、JPEG、PNG、PDF，文件大小不超过10MB。

#### 操作流程

1. 进入入住办理页面，点击发起申请按钮
2. 填写老人基本信息（姓名、身份证号、联系方式等）
3. 选择楼层、房间、床位
4. 选择护理等级
5. 关联合同信息
6. 上传必要文档
7. 提交申请

#### 相关接口

- POST /check-in/apply：发起入住申请
- GET /check-in/pageQuery：分页查询入住申请
- GET /check-in/detail/{id}：获取入住详情
- POST /check-in/upload：上传入住文档

### 老人管理

老人管理模块维护养老机构内所有老人的基本信息和入住状态。

#### 功能入口

入住信息查询：查看指定老人的详细入住信息，包括床位、护理等级、护理员等。

设置护理员：为老人分配或更换负责的护理员。

#### 操作流程

1. 进入老人管理页面，查看老人列表
2. 使用搜索条件筛选目标老人
3. 点击查看详情获取老人入住信息
4. 通过设置护理员功能分配护理人员

#### 相关接口

- GET /elder/selectList：获取老人列表
- GET /elder/pageQuery：分页查询老人
- GET /elder/selectByIdCard：身份证号校验
- GET /elder/checkInInfo/{id}：查询入住信息
- PUT /elder/setNursing：设置护理员

### 床位管理

床位管理模块负责楼层、房间、床位的层级管理和状态维护。

#### 功能入口

菜单路径：入住管理 > 床位管理

#### 主要功能

楼层管理：添加、编辑、删除楼层信息，查看楼层下所有房间和床位。

房间管理：在指定楼层下添加房间，编辑房间信息（房型、面积等），删除空闲房间。

床位管理：在房间内添加床位，编辑床位信息，删除空闲床位，查看床位状态。

房型设置：管理房型配置，支持启用/禁用房型。

#### 数据结构

系统采用楼层-房间-床位三级结构：
- 楼层：包含多个房间
- 房间：属于某个楼层，关联房型，包含多个床位
- 床位：属于某个房间，有独立状态（空闲、占用等）

#### 操作流程

添加床位：
1. 进入床位管理页面
2. 选择目标楼层和房间
3. 点击添加床位
4. 填写床位编号等信息
5. 提交保存

#### 相关接口

- GET /bed/read/room/{roomId}：根据房间查询床位
- POST /bed/create：新增床位
- GET /bed/read/{id}：查询床位详情
- PUT /bed/update：更新床位
- DELETE /bed/delete/{id}：删除床位
- POST /room/add：添加房间
- PUT /room/update：更新房间
- DELETE /room/delete/{id}：删除房间
- POST /floor/add：添加楼层
- PUT /floor/update：更新楼层
- DELETE /floor/delete/{id}：删除楼层

### 预约管理

预约管理模块处理客户的参观预约和来访登记。

#### 功能入口

菜单路径：预约管理 > 预约列表 / 来访管理 > 来访列表

#### 主要功能

新增预约：创建新的参观预约，记录预约人信息、预约时间、预约类型。

预约列表：分页查询预约记录，支持按姓名、手机号、状态、类型、时间范围筛选。

预约报到：预约人到院后执行报到操作，记录实际到访时间。

取消预约：取消已创建的预约记录。

删除预约：删除预约记录。

时间段余量查询：查询各时间段剩余可预约次数。

#### 预约状态

- 待确认：新创建的预约
- 已确认：确认有效的预约
- 已完成：已报到的预约
- 已取消：用户主动取消的预约

#### 操作流程

创建预约：
1. 进入预约管理页面
2. 点击新增预约
3. 填写预约人姓名、手机号
4. 选择预约日期和时间段
5. 选择预约类型（参观、咨询等）
6. 提交保存

预约报到：
1. 在预约列表中找到目标预约
2. 点击报到按钮
3. 系统记录当前时间为到访时间

#### 相关接口

- POST /reservation：新增预约
- PUT /reservation/{id}：更新预约
- DELETE /reservation/{id}：删除预约
- PUT /reservation/{id}/cancel：取消预约
- PUT /reservation/{id}/visit：预约报到
- GET /reservation/page：分页查询预约
- GET /reservation/countByTime：查询时间段余量
- GET /reservation/cancelled-count：获取取消次数

### 智能助手

智能助手是基于RAG技术的知识问答系统，为工作人员提供操作导航、制度查询、流程指导等智能问答服务。

#### 功能入口

菜单路径：智能助手 > 知识问答 / 知识库管理 / 命中详情

#### 知识问答

知识问答页面是智能助手的主要交互界面。

功能说明：
- 对话区：展示用户提问和AI回答的完整对话记录
- 输入框：支持输入问题，按Enter键发送，Shift+Enter换行
- 推荐问题：系统预设的常见问题，点击可直接提问
- 会话历史：左侧面板展示历史会话列表，支持切换和删除
- 新建会话：创建新的对话会话
- 回答状态：每条AI回答下方展示置信度、采用片段数、问题类型等信息
- 命中详情：点击查看命中详情可了解回答引用的知识库片段
- 重试：回答失败时可点击重试按钮重新生成

问题类型：
- 事实型：查询具体事实信息
- 流程型：查询操作流程和步骤
- 列表型：查询列表类信息
- 比较型：对比不同选项
- 条件型：基于条件的判断回答

操作流程：
1. 进入智能助手页面
2. 在输入框中输入问题
3. 按Enter键或点击发送按钮
4. 查看AI回答
5. 如需查看回答依据，点击命中详情
6. 可点击推荐问题快速体验

#### 知识库管理

知识库管理页面用于监控知识库状态和管理文档索引。

功能说明：
- 索引状态：展示知识库当前索引状态（可用、重建中、失败、未初始化）
- 文档统计：显示文档总数、分片总数、最近索引时间
- 文档列表：展示所有已导入知识库的文档，包含标题、分类、来源文件、更新时间、切片数、状态
- 失败记录：展示文档导入或索引失败的任务记录
- 重建索引：手动触发知识库索引重建
- 刷新：刷新页面数据

索引状态说明：
- 可用：索引正常，可正常问答
- 重建中：正在重建索引，期间可能影响问答质量
- 失败：索引构建失败，需查看失败记录并处理
- 未初始化：知识库尚未导入文档

操作流程：
重建索引：
1. 进入知识库管理页面
2. 查看当前索引状态
3. 如需更新知识，点击重建索引按钮
4. 等待索引重建完成
5. 重建完成后刷新页面确认状态

#### 命中详情

命中详情页面展示AI回答时检索到的知识库片段。

功能说明：
- 已采用片段：被AI回答实际引用的知识库片段
- 已丢弃片段：检索到但未被采用的候选片段
- 片段信息：展示文档标题、来源文件、综合评分、匹配关键词、片段内容
- 统计信息：采用片段数、候选片段总数、置信度

#### 智能助手技术架构

系统采用三层架构：
- 前端层：Vue3页面负责对话交互、会话管理、流式展示
- Java网关层：Spring Boot负责登录态校验、权限控制、请求转发、审计记录
- Python AI层：LangChain负责文档解析、向量检索、提示词编排、回答生成

检索策略：
- 向量检索：使用Embedding将问题和文档映射到向量空间进行相似度匹配
- 关键词召回：作为兜底策略补充向量检索
- 多路召回：综合多种检索结果提升召回质量
- 重排序：对召回结果进行精排筛选

回答生成约束：
- 仅依据知识库内容回答
- 不确定时明确告知知识库暂无依据
- 对系统导航类问题优先输出菜单路径和操作步骤
- 不编造知识库中不存在的信息

## 系统配置

### 用户权限

系统采用基于角色的访问控制（RBAC）：
- 用户：系统使用者，关联角色和部门
- 角色：权限集合，关联资源菜单
- 资源：系统菜单和操作权限
- 部门：组织架构，用于数据权限隔离
- 岗位：用户岗位信息

### 登录认证

- 用户名密码登录
- JWT Token认证
- Token有效期配置
- 登录态通过UserThreadLocal传递

### 文件存储

- 使用阿里云OSS对象存储
- 支持图片、PDF等文件格式
- 文件大小限制：10MB

## 常见问题

### 知识库暂无依据怎么办

当智能助手回答知识库暂无依据时，说明当前问题在知识库文档中没有相关内容。处理方式：
1. 确认问题表述是否准确，尝试换一种提问方式
2. 检查知识库是否包含相关文档
3. 如需补充知识，联系管理员在知识目录添加相关文档
4. 添加文档后在知识库管理页面重建索引

### 索引重建失败如何处理

1. 查看失败记录中的失败原因
2. 检查知识目录下的文档格式是否正确
3. 确认文档内容无乱码、格式异常
4. 移除问题文档后重新触发重建

### 护理任务无法执行

1. 确认任务状态是否为待执行
2. 确认当前用户是否为任务分配的护理员
3. 检查任务是否已过期
4. 如任务时间不合适，使用改期功能调整执行时间

### 床位无法删除

床位删除失败通常是因为床位已被占用：
1. 确认床位状态是否为空闲
2. 如床位有入住老人，需先办理退住
3. 退住完成释放床位后再执行删除

### 护理等级在哪里配置

菜单路径：服务管理 > 护理等级

操作步骤：
1. 进入服务管理模块
2. 点击左侧菜单护理等级
3. 在护理等级列表页面可查看所有护理等级
4. 点击新增护理等级按钮创建新的护理等级
5. 填写等级名称、服务内容、收费标准等信息后保存

### 老人入住办理需要哪些步骤

入住办理完整流程：
1. 进入入住管理 > 入住办理页面
2. 点击发起入住申请按钮
3. 填写老人基本信息（姓名、身份证号、联系方式等）
4. 选择楼层、房间、床位
5. 选择护理等级
6. 关联合同信息
7. 上传必要文档（身份证、体检报告等）
8. 提交申请等待审核

### 护理计划怎么创建

菜单路径：服务管理 > 护理计划

操作步骤：
1. 进入服务管理模块，点击护理计划
2. 在护理计划列表页面点击新增护理计划
3. 填写计划名称、计划说明
4. 选择关联的护理项目（可多选）
5. 设置各护理项目的执行频率和时间
6. 选择适用的护理等级或老人
7. 保存护理计划

### 退住办理流程是什么

菜单路径：入住管理 > 退住管理

操作步骤：
1. 进入入住管理模块，点击退住管理
2. 点击发起退住申请按钮
3. 选择需要退住的老人
4. 填写退住原因
5. 选择预计退住时间
6. 提交退住申请
7. 系统生成退住结算单
8. 完成费用结算后释放床位

### 预约来访如何登记

菜单路径：预约管理 > 预约列表

操作步骤：
1. 进入预约管理模块，点击预约列表
2. 点击新增预约按钮
3. 填写预约人姓名、手机号
4. 选择预约日期和时间段
5. 选择预约类型（参观、咨询等）
6. 填写来访人数和备注信息
7. 提交保存预约记录

预约报到：
1. 在预约列表中找到已确认的预约
2. 点击报到按钮
3. 系统自动记录当前时间为实际到访时间

### 智能床位能查看哪些数据

菜单路径：智能床位

智能床位可查看的数据包括：
1. 床位实时状态：空闲、占用、预约等
2. 入住老人信息：姓名、年龄、护理等级
3. 智能设备数据：
   - 床垫压力分布
   - 心率监测数据
   - 呼吸频率
   - 离床报警记录
   - 睡眠质量分析
4. 护理任务提醒：待执行的护理任务
5. 历史数据趋势：各项健康指标的变化趋势

### 账单入账列表在哪里

菜单路径：财务管理 > 账单管理 > 入账列表

操作说明：
1. 进入财务管理模块
2. 点击左侧菜单账单管理
3. 选择入账列表标签页
4. 可查看所有入账记录，支持按时间范围、老人姓名、账单类型筛选
5. 支持导出入账明细报表

## 知识目录结构

系统知识库采用以下目录结构组织文档：

- knowledge-base/
  - system-manuals/：系统操作手册
  - care-specs/：护理规范文档
  - process-guides/：业务流程指南
  - faq/：常见问题解答

文档导入由管理员在知识目录下放置文档，Python服务扫描目录并生成索引。
