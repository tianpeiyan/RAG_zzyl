**总目标**
在现有养老系统中上线一个可用的 RAG 智能助手 V1，支持文档问答、操作导航、基础知识库管理，具备最小可上线能力。

**总体验收指标**

- 知识库问答可用率：>= 95%
- 典型问题回答正确率：>= 80%
- 文档导入成功率：>= 95%
- 问答接口 P95 响应时长：<= 8s
- 未命中问题误答率：<= 10%
- 前端关键流程可用性：100%，即“能进入页面、能提问、能得到结果、能看到错误提示”

**阶段计划**

1. 阶段一：项目基线与验收样本建立
   执行内容：明确目录结构、服务边界、接口清单，准备首批知识文档和 30 条验收问题集。问题集至少覆盖“护理规范、系统操作、流程指导、无答案问题”四类。
   交付物：架构图、接口草案、知识文档目录规范、验收问题清单。
   通过指标：30 条问题样本整理完成；文档分类不少于 3 类；所有验收问题都能映射到“应答/拒答”预期结果。
2. 阶段二：Python LangChain 服务骨架搭建
   执行内容：新建独立 AI 服务，完成配置管理、日志、健康检查、内部 API 骨架，接入通义 LLM 与 Embedding。
   交付物：Python 服务可启动；具备 /internal/chat、/internal/kb/reindex、/internal/kb/status、/internal/kb/documents 基础接口。
   通过指标：本地启动成功；健康检查连续 10 次请求无失败；配置项可通过环境变量切换；接口联调返回成功率 100%。
3. 阶段三：知识库导入与索引链路实现
   执行内容：完成 PDF、DOCX、MD、TXT 文档解析，做清洗、切片、Embedding、Chroma 持久化，支持固定目录全量重建。
   交付物：导入脚本、索引构建模块、文档元数据结构。
   通过指标：首批文档导入成功率 >= 95%；单次全量重建成功率 100%；重建后能查到对应文档；异常文档会被记录失败原因，不允许静默失败。
4. 阶段四：RAG 问答链路实现
   执行内容：完成“问题预处理 -> 检索 -> 上下文拼装 -> 大模型生成 -> 拒答控制”完整链路，并针对“系统功能导航”补充固定提示词模板。
   交付物：问答服务主流程、提示词模板、拒答规则。
   通过指标：在 30 条验收问题上，正确率 >= 70%；无答案问题拒答率 >= 80%；明显幻觉回答数 <= 3 条；P95 响应时长 <= 8s。
5. 阶段五：Java 网关/BFF 集成
   执行内容：在现有 Spring Boot 中新增 /api/assistant/* 对外接口，完成登录态复用、权限校验、请求转发、异常处理、审计日志。
   交付物：Java 代理接口、审计日志记录、统一错误码。
   通过指标：前端只调用 Java 即可完成全链路问答；未登录访问被正确拦截；Python 服务异常时前端能收到可读错误；审计日志覆盖率 100%，即每次提问均有记录。
6. 阶段六：前端智能助手页面落地
   执行内容：在 Vue3 管理端新增独立“智能助手”页面，完成输入框、对话区、加载态、空态、错误态、推荐问题。
   交付物：问答页、菜单入口、接口适配。
   通过指标：完成 5 个核心交互场景测试通过：
   - 进入页面
   - 发起提问
   - 查看回答
   - 处理加载中
   - 处理失败重试
     页面无阻塞性报错；连续提问 20 次无前端崩溃。
7. 阶段七：知识库基础管理页
   执行内容：新增管理页面，支持查看文档列表、索引状态、手动重建、失败记录。首版不做上传，只做运营可见性。
   交付物：知识库管理页、状态接口联动。
   通过指标：运营人员可独立完成一次索引重建；文档列表展示字段完整；失败记录可追踪到具体文件；重建状态刷新准确率 100%。
8. 阶段八：联调优化与效果校准
   执行内容：基于验收问题集反复优化切片大小、召回条数、提示词和拒答阈值，修正高频误答。
   交付物：优化后的参数配置、问题评估报告。
   通过指标：30 条样本整体正确率提升到 >= 80%；无答案误答率降到 <= 10%；高频问题平均响应时长 <= 6s；重复问题回答稳定性 >= 90%。
9. 阶段九：预上线与试运行
   执行内容：部署测试环境，安排真实用户试用，收集问题日志和失败案例，完成上线检查清单。
   交付物：试运行报告、上线清单、风险清单。
   通过指标：试运行期间核心功能可用率 >= 95%；严重故障 0 个；收集不少于 20 条真实问题反馈；上线阻断项全部关闭。

**推进判定规则**

- 每个阶段必须同时满足“交付物完成 + 指标达标”才能进入下一阶段。
- 如果阶段指标未达标，只允许在本阶段内优化，不跳阶段。
- 每周至少输出一次阶段报告，报告字段固定为：已完成项 / 当前指标 / 未达标项 / 下周动作 / 风险



**里程碑判定**

- 里程碑 M1：Python 知识导入成功，说明“知识库已可建立”。
- 里程碑 M2：Java 成功代理聊天请求，说明“系统已可接入”。
- 里程碑 M3：前端聊天页可稳定提问，说明“用户已可使用”。
- 里程碑 M4：知识库管理页可重建索引，说明“系统已可运维”。
- 里程碑 M5：标准问题集达标，说明“系统具备上线条件”。
