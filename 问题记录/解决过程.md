当前 RAG 智能助手的根本问题是：检索相关性判断主要依赖人工规则和弱语义信号，而不是稳定、可泛化的语义匹配与重排判断。

## 2026-04-26 修复记录

1. 将生产默认 embedding provider 从 `local` 调整为 `dashscope`，`.env.example` 使用 `dashscope-multimodal`，保留 local hash embedding 仅用于离线测试和本地调试，避免作为生产检索主力。
2. 在混合召回后增加 rerank 阶段，召回候选仍按 `rerank_candidate_k` 保持宽召回，最终进入 LLM 上下文前再结合 rerank 分、向量分、关键词分重新排序。
3. 增加最终上下文门控：`final_context_min_score`、`short_query_final_context_min_score`、`answerability_min_score`，短问题采用更高阈值。
4. 增加 answerability 判断，片段必须覆盖关键问题词、具备直接回答动作意图的证据，且可回答性分达到阈值，才会被标记为 `selected=true` 并进入上下文。
5. 降低泛词命中影响，对“养老、老人、服务、哪些、什么”等低信号词命中做惩罚，避免只靠泛词进入最终答案。
6. 增加通用 query focus/action 抽取，输出 `focus_terms` 与 `action_terms`，减少通过 `_term_aliases` 反复补词来修正个案的依赖。
7. 接口测试补充覆盖：短位置问题过滤泛词片段、配置类问题拒绝不可回答片段、配置证据命中、rerank 分参与最终采用、默认生产 embedding provider 为语义 embedding。
