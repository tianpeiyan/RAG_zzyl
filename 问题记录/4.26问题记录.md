4、意图识别过硬，关键词匹配粗糙



### 缺乏问题类型细分
当前只有三种意图： SMALL_TALK / DOMAIN_QA / OUT_OF_SCOPE 。

缺少：

- 事实型问答 （如"护理等级在哪里配置？"）

- 流程型问答 （如"退住办理流程是什么？"）

- 列表型问答 （如"智能床位能查看哪些数据？"）

- 比较型/条件型问答
  不同类型的回答策略应该不同——流程型问题需要分步骤回答，而不是罗列相关片段。

  

#### 解决方案：

保留 SMALL_TALK / DOMAIN_QA / OUT_OF_SCOPE，在 DOMAIN_QA 下增加 FACT / PROCESS / LIST / COMPARISON / CONDITION，用规则词表先覆盖“流程、步骤、哪些、区别、条件、如何”等高频问法



5、RAG检索与召回缺陷

​	检索不精准，用户问退住办理流程，返回了两个不相关片段，唯一相关的还不是退住流程而是退住规则

![image-20260426143024665](C:\Users\lenovo\AppData\Roaming\Typora\typora-user-images\image-20260426143024665.png)

原因分析：
 - langchain_vector_store.py的 similarity_search 基于向量相似度，"退住"和"入住"、"退费"在语义空间可能很近

#### 解决方案

-  查询重写/扩展 （Query Rewriting）：将"流程"这个关键需求传递给检索系统
-  混合检索 （BM25 + 向量）：纯向量检索对特定术语（如"流程"）不敏感



检索链路改为“查询分析 + 查询扩展 + 混合召回 + 重排”：

- 对“退住办理流程是什么”生成检索词：原问题、退住 流程 步骤 办理、退住办理。
- 在现有向量召回基础上增加轻量 BM25/关键词召回，不新增重依赖，优先匹配“退住 + 流程/办理/步骤”这类强关键词。
- 合并召回结果后按综合分排序，并记录每个片段的 vectorScore / keywordScore / finalScore / matchedTerms / selected。
